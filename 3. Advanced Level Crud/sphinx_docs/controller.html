controller module¶

Uso de módulos de ‘tkinter’.

Para este módulo, desde ‘tkinter’ se realiza la importación de ‘Tk’.

controller.py:

Módulo que realiza la comunicación entre los archivos ‘view.py’ y ‘model.py’ al momento que el usuario, mediante la ejecución de eventos definidos en el CRUD, solicita información a la aplicación. En síntesis, el ‘controlador’ es el cerebro que controla y decide cómo mostrar los datos, iniciando la ‘vista’ (view.py) y modificando el ‘modelo’ (model.py).

class controller.Controlador(root_w)[source]¶

Bases: object

Se define la clase ‘Controlador’ y se utiliza el método ‘__init__’ para iniciar los atributos de los objetos creados. Además, introduce el patrón UML (patrón Observador).

Parameters

root_w – base de la interfaz gráfica (raíz, contenedor base de los widgets que forman la interfaz).
